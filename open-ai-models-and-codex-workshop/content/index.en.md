Welcome to this workshop on OpenAI's GPT models and OpenAI Codex, all running on Amazon Bedrock. Bedrock exposes an OpenAI-compatible endpoint, so you can write the same OpenAI SDK and Responses API code you would use against OpenAI directly. 

## Part 1: OpenAI Frontier Models with the Responses API

Part 1 uses GPT-5.5 through the Responses API on Bedrock by default. You will:

- Make direct API calls with curl and the OpenAI SDK
- Use function calling and structured outputs
- Process image inputs
- Build a small agent workflow

## Part 2: OpenAI Codex App, IDE, and CLI

In Part 2 you configure OpenAI Codex to run its inference on Bedrock. You will:

- Point the same Bedrock credentials and endpoint at the Codex App, IDE extension, and CLI
- Work hands-on primarily in the Codex App, the recommended surface
- Optionally use the IDE and CLI paths if you prefer those workflows

## Learning Objectives

By the end of this workshop, you will be able to:

- Call OpenAI GPT models on Amazon Bedrock using ordinary OpenAI SDK and Responses API code
- Build real workflows with function calling, structured outputs, images, and agents
- Run and govern OpenAI Codex on Bedrock across the App, IDE, and CLI
